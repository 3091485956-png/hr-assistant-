Chapter I

诚实：这个项目是怎么诞生的

我首先需要诚实地说：这个项目里的绝大多数代码，并不是我亲手一行一行敲出来的。

我是英语专业的大二学生，我不会写代码。在决定做这个项目之前，我甚至不知道 React 和 TypeScript 是什么。但我会做一件事——提出问题，验收答案，理解逻辑。

这个项目是我和 AI 共同完成的。我负责定义需求、描述场景、提出设想、判断好坏；AI 负责把人的语言翻译成机器的语言，把模糊的意图变成精确的代码。在这个意义上，我更像是一个产品负责人、一个验收者、一个用 AI 做事的人。

"这不是什么需要隐瞒的事。恰恰相反，这正是我最想说的：在一个 AI 可以编程的时代，人还能做什么？人的价值在哪里？"
Chapter II

自造之路：一个文科生的技术远征

我必须诚实地说，在开始做这个项目之前，我连 HTML 是什么都不知道。 div 和 class 对我来说，就是两个陌生的单词拼接在一起。

但我始终相信一件事：学习本身就是天赋，而跨界的勇气比天赋更珍贵。

看见问题

在 HR 实习中，看到每天上百份简历堆在邮箱里，HR 手动录入、筛选、写邮件，大量时间被机械劳动吃掉。

提出问题

如果 AI 能读懂莎士比亚的十四行诗，它能不能读懂一份简历里藏着的、那个人尚未言说的可能性？

定义产品

设计四大功能模块：简历入库、岗位匹配、人才库管理、AI 话术生成。每一个功能都对应一个真实的 HR 痛点。

与 AI 协作

从零开始描述需求，验收每一版代码，调试每一个 bug，直到一个完整的产品诞生在屏幕上。

反思意义

不只是做产品，而是想清一个问题：当 AI 可以做越来越多的事时，人还剩下什么不可替代的价值？

Philosophy

存在主义之问：AI 带来了人的"存在危机"吗？

做这个项目的过程中，我一直在想这个问题。如果 AI 可以帮我做一个 HR 管理的软件，那未来 AI 是不是可以直接替代 HR？如果 AI 可以读简历、匹配岗位、生成话术，那 HR 这个职业还有什么存在的必要？

"AI 能读简历，但它不能'看见人'。"

一份简历在 AI 眼里是一堆关键词的集合——学历、技能、工作年限。但同样一份简历，在一个优秀的 HR 眼里，是一个人的故事。这个人为什么转行？为什么每段工作都做不满两年？这些问题的答案，不在数据里，在人的经验、直觉和共情能力里。

"AI 能生成话术，但它不能建立信任。"

一段完美的面试邀请话术，可以写得滴水不漏，但它无法传递一个 HR 对候选人的真诚关心。人不是被话术打动的，人是被另一个真实的人的温度打动的。

"AI 能匹配分数，但它不能承担选择的重量。"

当 AI 说"匹配度 85 分"时，这个分数是冰冷的。但 HR 知道，选择一个人、放弃一个人，背后是一个个真实的人生。这种道德重量，只有人才能承担。

Manual

使用指南：从零开始到成功运行

如果你完全不懂技术，不用担心。下面的每一步都经过精心拆解，你只需要跟着做，就能在自己的电脑上运行这个软件。

1

下载项目

将项目从 GitHub 下载到你的电脑

访问项目页面

用浏览器打开 GitHub 仓库地址：

URL https://github.com/3091485956-png/hr-assistant-

下载压缩包

  1. 在页面右侧找到绿色的 <> Code 按钮，点击它
  2. 在弹出菜单中选择 Download ZIP
  3. 浏览器会下载一个名为 hr-assistant-.zip 的压缩包

解压到桌面

  1. 打开"下载"文件夹，找到 hr-assistant-.zip
  2. 右键点击 -> 选择"全部解压缩"（Windows 11）或"解压到当前位置"
  3. 将解压后的 hr-assistant- 文件夹拖动到桌面
2

安装 Node.js

运行环境只需要安装一次，以后不用再装

下载并安装

访问 nodejs.org，找到绿色的 LTS 按钮（写着 "Recommended For Most Users"），点击下载。

💡
安装提示

Windows 下载 .msi 文件，Mac 下载 .pkg 文件。双击安装后一直点 Next，所有选项保持默认，最后点 Finish。

确认安装成功

重新打开一个新的终端窗口：

  • Windows：按 Win + R，输入 cmd，按回车
  • Mac：打开"启动台" -> 搜索"终端" -> 打开

在终端中依次输入：

Terminal node -v
npm -v

如果都显示版本号（如 v20.x.x），说明安装成功。

3

安装项目依赖

让软件自动下载所需的库文件

进入项目文件夹

在终端中输入以下命令，进入桌面上的项目文件夹：

Terminal cd Desktop\hr-assistant-
⚠
注意

如果你的文件夹名字不同（比如有版本号后缀），请把命令中的 hr-assistant- 替换成你实际的文件夹名字。

执行安装命令

Terminal npm install

按回车后，屏幕会出现大量滚动的文字，这是在下载各种依赖文件。请耐心等待，不要关闭窗口。 这个过程通常需要 3 到 10 分钟。

当看到类似 added xxx packages in xxm 的输出时，说明安装完成。

🔧
卡住怎么办

如果超过 15 分钟没有任何动静，关掉终端重新打开，重新运行 npm install。也可以先切换国内镜像源：npm config set registry https://registry.npmmirror.com

4

创建数据库

SQLite 本地数据库，无需安装额外软件

同步数据库结构

在终端中输入：

Terminal npm run db:push

当看到 [✓] Changes applied 时，数据库就已准备就绪。这一步只需要执行一次。

5

启动项目

让软件在你的电脑上运行起来

运行开发服务器

Terminal npm run dev

当看到以下输出时，说明软件已经成功运行：

Output VITE v6.x.x ready in xxx ms

➤ Local: http://localhost:3000/

在浏览器中打开

打开 Chrome、Edge 或 Safari，在地址栏输入：

URL http://localhost:3000

按回车，你就会看到 HR辅助器 的登录界面了！

FAQ

常见问题

遇到问题不要慌，先看看这里有没有答案。

终端显示"npm 不是内部或外部命令" +

Node.js 没有安装成功。请回到"第二步"重新安装，安装完成后务必重新打开终端窗口。

npm install 很慢或者卡住 +

这是网络问题。关掉终端重新打开，再次运行 npm install。如果仍然失败，先运行 npm config set registry https://registry.npmmirror.com 切换国内镜像源，然后再安装。

npm run db:push 报错 +

检查是否进入了正确的文件夹（路径最后应该是 hr-assistant-），然后重新运行。如果仍然报错，确认前面的 npm install 已经成功完成。

怎么关闭软件 +

回到终端窗口，按 Ctrl + C，然后输入 Y 按回车确认。

下次怎么重新打开软件 +

重新打开终端，输入以下命令：
cd Desktop\hr-assistant-
npm run dev
然后浏览器访问 http://localhost:3000

配置 AI 时"测试连接"失败 +

检查三点：1) API Base URL 是否填写正确（注意末尾的 /v1）；2) API Key 是否复制完整，没有多余空格；3) 你的网络是否能正常访问该 AI 服务商。推荐使用 DeepSeek，国内访问最稳定。
